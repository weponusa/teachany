# 知识树状态字段修复报告

## 🐛 问题发现

用户反馈：**高中化学「氧化还原反应」课件已存在，但知识图谱中显示没有课件。**

## 🔍 问题定位

### 根本原因

1. **课件确实存在**：
   - 路径：`examples/chem-oxidation-reduction/`
   - 已注册到 `registry.json` 和 `courseware-registry.json`
   - 已在 `chemistry-high.json` 知识树中正确配置

2. **状态字段不规范**：
   ```json
   {
     "id": "oxidation-reduction",
     "courses": ["chem-oxidation-reduction"],
     "status": "has_course"  // ❌ 错误！
   }
   ```

3. **规范冲突**：
   - `SKILL_CN.md` 第1394行规定：
     > 仅 `status: "active"` 且 `courses` 非空的节点为 `true`
   - 但所有知识树中使用的是 `"has_course"` 状态
   - 导致**所有已有课件的节点在知识图谱中都显示为没有课件**！

---

## 🔧 修复方案

### 批量修复脚本

创建 `scripts/fix-tree-status.py`：

```python
#!/usr/bin/env python3
"""
批量修复知识树中的状态字段
将 "status": "has_course" 统一改为 "status": "active"
"""

import json
from pathlib import Path

def fix_tree_status(tree_file):
    with open(tree_file, 'r', encoding='utf-8') as f:
        data = json.load(f)
    
    fixed_count = 0
    for domain in data.get('domains', []):
        for node in domain.get('nodes', []):
            if node.get('status') == 'has_course':
                node['status'] = 'active'
                fixed_count += 1
    
    if fixed_count > 0:
        with open(tree_file, 'w', encoding='utf-8') as f:
            json.dump(data, f, ensure_ascii=False, indent=2)
    
    return fixed_count
```

### 2. 验证机制

建议添加 CI 检查脚本：

```python
def validate_tree_status():
    """验证所有知识树节点的状态字段"""
    allowed_status = {"active", "gap", "deprecated"}
    
    for tree_file in Path("data/trees").glob("*.json"):
        tree = json.load(tree_file.open())
        for domain in tree["domains"]:
            for node in domain["nodes"]:
                status = node.get("status")
                if status not in allowed_status:
                    raise ValueError(
                        f"Invalid status '{status}' in {tree_file.name}:"
                        f" node {node['id']}"
                    )
```
