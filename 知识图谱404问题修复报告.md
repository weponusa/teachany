# 知识图谱404问题修复报告

**用户反馈**：

**症状**：

2. 点击 tooltip 中的"进入课件"按钮
3. 浏览器跳转到错误的URL，返回 404 Not Found
4. 影响所有 131 个课件的访问

## 🔍 问题诊断

```bash

```

**结论**：后端索引完全正常，问题出在前端。

### 前端问题定位

**错误的URL生成**（tree.html）：

```javascript
// ❌ 错误代码（第818行）
href="${d.courses[0]}/index.html"
// 生成: chn-compound-vowel/index.html (相对路径，404)

// ❌ 错误代码（第904行）
window.open(d.courses[0] + '/index.html', '_blank');
// 生成: chn-compound-vowel/index.html (缺少examples/前缀)
```

**正确的URL应该是**：
```
./examples/chn-compound-vowel/index.html
```

---

## 🔧 修复方案

### 修改文件：`tree.html`

#### 修复1：Tooltip中的链接（第818行）
```javascript
// 修复前
if (d.status === 'active' && d.courses.length) {
  html += `<a class="course-link" href="${d.courses[0]}/index.html" target="_blank">🎯 进入课件</a>`;
}

// 修复后
if (d.status === 'active' && d.courses.length) {
  html += `<a class="course-link" href="./examples/${d.courses[0]}/index.html" target="_blank">🎯 进入课件</a>`;
}
```

#### 修复2：节点点击事件（第904行）
```javascript
// 修复前
window.open(d.courses[0] + '/index.html', '_blank');

// 修复后
window.open('./examples/' + d.courses[0] + '/index.html', '_blank');
```

## 🎯 根本原因分析

### 为什么会出现这个问题？

1. **历史遗留代码**：
   - tree.html 中的课件链接代码可能是早期版本
   - 当时课件可能直接放在根目录
   - 后来迁移到 `examples/` 目录后，链接未更新

2. **测试覆盖不足**：
   - 首页卡片使用了正确的路径（`./examples/{id}/`）
   - 但知识图谱的链接逻辑未测试
   - 导致问题隐藏至今

3. **路径处理不一致**：
   - 不同页面使用不同的路径生成方式
   - 缺少统一的URL工具函数

---

## 💡 预防措施

### 1. 统一路径处理
建议创建统一的路径工具函数：
```javascript
// 建议在 scripts/unified-loader.js 中添加
function getCourseUrl(courseId) {
  return `./examples/${courseId}/index.html`;
}
```

### 2. 自动化测试
添加链接有效性测试脚本：
```bash
# 验证所有课件链接
python3 scripts/validate-courseware-links.py
```

### 3. 文档化
在 SKILL_CN.md 中明确说明：
- 所有课件必须放在 `examples/{course-id}/` 目录
- 所有课件链接必须使用 `./examples/{course-id}/index.html` 格式
- 禁止使用相对路径如 `{course-id}/index.html`
