# 🎯 TeachAny 国内教师安装方案（完整版）

---

## 📋 方案总览

为了让国内中小学教师能够**轻松、快速、无障碍**地使用 TeachAny，我们提供了**5种安装方式** + **自动化部署流程**。

### 核心问题
- ❌ GitHub 在国内访问不稳定，经常需要翻墙
- ❌ 技术门槛高，教师不熟悉 Git 操作
- ❌ 下载速度慢，影响使用体验

### 解决方案
- ✅ Gitee 镜像（国内高速访问）
- ✅ 百度网盘离线包（无需注册）
- ✅ 国内 CDN 加速（阿里云/腾讯云）
- ✅ 一键安装脚本（自动化安装）
- ✅ 详细的图文/视频教程

## 🚀 五种安装方式

### 方式一：Gitee 镜像（推荐）⭐⭐⭐⭐⭐

**适合人群**：所有用户  
**技术要求**：无  
**访问速度**：⚡⚡⚡⚡⚡

**步骤**：
1. 访问 https://gitee.com/wepon/teachany-opensource
2. 点击 **"克隆/下载"** → **"下载 ZIP"**
3. 解压到 `D:\TeachAny\`
4. 打开 CodeBuddy → 设置 → Skills 管理 → 导入本地 Skill

**优点**：
- 实时与 GitHub 主仓库同步
- 国内 CDN 加速，速度极快
- 无需翻墙，稳定可靠

**文档**：[INSTALL_CN.md](./INSTALL_CN.md)

---

### 方式二：百度网盘（最简单）⭐⭐⭐⭐

**适合人群**：完全不懂技术的教师  
**技术要求**：无  
**访问速度**：⚡⚡⚡⚡

**步骤**：
1. 访问百度网盘链接：https://pan.baidu.com/s/xxx（提取码：`ta66`）
2. 下载 `TeachAny-完整版-2026.04.14.zip`
3. 解压并按照压缩包内的 `安装说明.txt` 操作

**优点**：
- 无需注册 Gitee 账号
- 下载速度快（有会员更快）
- 适合集体分发（可分享链接给同事）

**缺点**：
- 更新频率：每月更新一次（不是实时）

### 方式三：一键安装脚本（自动化）⭐⭐⭐⭐⭐

**适合人群**：有一定技术基础的教师  
**技术要求**：会使用命令行/终端  
**访问速度**：⚡⚡⚡⚡⚡

#### macOS/Linux：

```bash
# 下载并执行安装脚本
curl -fsSL https://gitee.com/wepon/teachany-opensource/raw/main/install-cn-auto.sh | bash
```

#### Windows PowerShell：

```powershell
# 下载安装脚本
Invoke-WebRequest -Uri "https://gitee.com/wepon/teachany-opensource/raw/main/install-cn-auto.ps1" -OutFile "install.ps1"

# 执行安装（需要管理员权限）
Set-ExecutionPolicy Bypass -Scope Process -Force
.\install.ps1
```

**优点**：
- 全自动安装，无需手动操作
- 自动检查依赖
- 自动备份旧版本

## 🛠️ 自动化部署流程（给维护者）

### 步骤1：本地提交代码

```bash
cd teachany-opensource
git add .
git commit -m "feat: 更新教学数据"
```

### 步骤2：运行部署脚本

```bash
./deploy-cn.sh
```

**脚本自动完成**：
1. ✅ 推送到 GitHub（国际主仓库）
2. ✅ 同步到 Gitee（国内镜像）
3. ✅ 生成压缩包（用于百度网盘分发）
4. ✅ 生成部署报告

### 步骤3：上传到百度网盘

将生成的 `TeachAny-完整版-YYYY.MM.DD.zip` 手动上传到百度网盘，并更新分享链接。

### 步骤4：更新镜像站列表

编辑 `MIRRORS_CN.md`，更新版本号和下载链接。

### 步骤5：通知用户

在微信群/QQ群发布更新通知，附上下载链接和更新日志。

## 🔄 GitHub ↔ Gitee 自动同步

### GitHub Actions 工作流

我们配置了 GitHub Actions，每次推送到 `main` 分支时，会**自动同步**到 Gitee：

**配置文件**：`.github/workflows/sync-to-gitee.yml`

**触发条件**：
- 推送到 `main` 分支
- 手动触发（在 GitHub Actions 页面点击"Run workflow"）

**同步内容**：
- 全部代码
- 全部标签（Tags）
- 全部分支

**配置步骤**（首次设置）：

1. 在 Gitee 生成 SSH 密钥对：
   ```bash
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com" -f ~/.ssh/gitee_deploy_key
   ```

2. 在 Gitee 仓库设置中添加**部署公钥**（Deploy Key）：
   - 公钥内容：`cat ~/.ssh/gitee_deploy_key.pub`
   - 勾选 **"允许推送"**

3. 在 GitHub 仓库设置中添加 **Secret**：
   - Name: `GITEE_SSH_PRIVATE_KEY`
   - Value: `cat ~/.ssh/gitee_deploy_key`（私钥内容）

4. 推送代码后，在 GitHub Actions 页面查看同步状态。
