# TeachAny Skill v5.12 更新总结

**更新日期**: 2026-04-12  
**核心变更**: ⭐ 强制使用开源数据源，禁止手工标注

---

## 一、更新动机

在制作历史/地理课件时发现手工标注存在严重问题:
- ❌ 河流路径低精度，误差大
- ❌ 城市坐标无出处，难以验证
- ❌ 战役地点随意标注，与史料不符
- ❌ 数据无法复用，每次都要重新手工输入

**解决方案**: 强制使用权威开源数据集（CHGIS V6 + Natural Earth）。

**数据处理流程**:
```bash
# 1. 下载 Shapefile → 2. ogr2ogr 转 GeoJSON → 3. mapshaper 简化 → 4. 存入项目
ogr2ogr -f GeoJSON output.json input.shp
mapshaper input.json -simplify 0.1% -o output-simplified.json
```

**GeoJSON metadata 标准字段**（强制）:
```jsonc
{
  "type": "FeatureCollection",
  "metadata": {
    "dataSource": "CHGIS V6",
    "sourceUrl": "https://dataverse.harvard.edu/dataset.xhtml?persistentId=...",
    "chgisId": "v6_1820_county_pgn.123", // 原始数据 ID
    "processedBy": "TeachAny",
    "processedDate": "2026-04-12"
  },
  "features": [...]
}
```

### 2.2 新增章节：Section 18.4.7《数据预处理工具链》

**工具组合**:
- **ogr2ogr** (GDAL): 格式转换 Shapefile → GeoJSON
- **mapshaper**: 几何简化（减少 90% 点数，视觉无损）

**在线工具**:

- **GeoJSON.io**: https://geojson.io/ (查看/编辑 GeoJSON)

**CHGIS 数据下载指南**:
1. 访问 https://dataverse.harvard.edu/dataverse/chgis_v6
2. 选择时间切片（如 `CHGIS V6 Qin (221 BC)`）
3. 下载 Shapefile 压缩包
4. 使用 ogr2ogr 转换为 GeoJSON
5. 存储到 `data/geography/historical-china/` 目录

**Natural Earth 数据下载**:

### 2.4 更新：Section 18.4.6《验收标准》

## 三、禁止与正确示例

### ❌ 禁止的做法

```javascript
// 错误 1: 手工标注坐标
const rivers = [
  { name: "黄河", path: [[110, 35], [112, 36], ...] } // 精度低，无出处
];

// 错误 2: 使用闭源 API
fetch('https://api.某商业地图.com/rivers?key=xxx') // 无法离线使用

// 错误 3: 直接嵌入大量 GeoJSON 到 HTML
<script>const data = {"type":"FeatureCollection","features":[...]}</script> // 文件巨大
```

### ✅ 正确的做法

```javascript
// 正确: 从预处理的 Natural Earth 数据加载
fetch('/data/geography/rivers/ne_10m_rivers_china.json')
  .then(res => res.json())
  .then(rivers => {
    map.addSource('rivers', {
      type: 'geojson',
      data: rivers
    });
    map.addLayer({
      id: 'rivers',
      type: 'line',
      source: 'rivers',
      paint: {
        'line-color': '#1976d2',
        'line-width': ['get', 'strokeweig'] // 使用 Natural Earth 自带属性
      }
    });
  });
```

## 六、常见问题 FAQ

### Q1: CHGIS 数据太大怎么办?
**A**: 使用 mapshaper 简化到 0.05%-0.1%，或按需裁剪特定区域:
```bash
mapshaper input.json -simplify 0.1% keep-shapes -o output.json
```

### Q2: 如何处理历史战役地点（CHGIS 没有）?
**A**: 从 CHGIS Place Names 数据库中查找对应古代地名坐标，手工整理到 `battles.json`，但必须标注 `chgisId` 字段作为溯源。

### Q3: 如何验证坐标准确性?
**A**: 在 https://geojson.io/ 中查看，对比现代卫星地图和历史地图集（如《中国历史地图集》）。

### Q4: Natural Earth 如何按国家过滤?
**A**: 使用 mapshaper 的 `-filter` 命令:
```bash
mapshaper ne_10m_rivers_lake_centerlines.json -filter 'SOV_A3 === "CHN"' -o ne_10m_rivers_china.json
```

### Q5: 如何处理缺失的数据（如某朝代的疆域）?
**A**: 
1. 优先从 CHGIS V6 查找对应时间切片
2. 如无现成数据，基于《中国历史地图集》手工数字化（使用 GeoJSON.io）
3. 在 metadata 中注明 `dataSource: "中国历史地图集第X册 + 手工数字化"`

```javascript
// 旧版（禁止）

```
