## 原因分析

### 为什么会出现重复节点？

1. **历史遗留问题**：多次手动编辑知识树时产生的冗余节点
2. **合并冲突**：Git 合并时未正确处理重复内容
3. **命名不统一**：早期 `compound-vowels`（复数）与后期 `compound-vowel`（单数）不一致
4. **概念 vs 课件混淆**：`element-concept`（宽泛概念）与 `periodic-table`（具体课件）未明确区分

### 为什么之前的工具未发现？

1. **rebuild-index.py 的局限性**：
   - 只能检测**相同 ID 的重复节点**
   - 无法检测**不同 ID 但引用同一课件**的情况

2. **diagnose-courses.py 的盲区**：
   - 只统计课件是否被引用
   - 不检查一个课件是否被**多个节点**引用

## 后续改进建议

### 1. 增强 rebuild-index.py 的检测能力

在 `rebuild-index.py` 中添加"同课件多节点"检测：

```python
def detect_duplicate_course_references(trees_dir):
    """检测同一课件被多个节点引用的情况"""
    course_to_nodes = defaultdict(list)
    
    for tree_file in glob.glob(f"{trees_dir}/*.json"):
        tree_data = json.load(open(tree_file, encoding='utf-8'))
        def extract(node, path=""):
            node_id = node.get('id', '')
            courses = node.get('courses', [])
            for course in courses:
                course_to_nodes[course].append({
                    'tree': os.path.basename(tree_file),
                    'node_id': node_id,
                    'path': path
                })
            # 递归处理子节点...
        extract(tree_data)
    
    # 报告重复引用
    duplicates = {c: nodes for c, nodes in course_to_nodes.items() if len(nodes) > 1}
    if duplicates:
        print("\n⚠️  发现同课件多节点引用:")
        for course, nodes in duplicates.items():
            print(f"\n  课件: {course}")
            for node in nodes:
                print(f"    - 节点ID: {node['node_id']} ({node['tree']})")
```

### 3. 知识树编写规范

**命名约定**：
- 节点 ID 使用**单数形式**（如 `compound-vowel` 而非 `compound-vowels`）
- 节点 ID 应与课件 manifest.json 中的 `node_id` 字段**完全一致**

**节点类型区分**：
- **概念节点**（`status: "gap"`）：不包含 `courses` 数组，用于组织知识结构
- **课件节点**（`status: "active"`）：包含 `courses` 数组，指向具体课件

**关系表达**：
- 概念节点应通过 `extends` 或 `parallel` 指向具体课件节点
- 一个课件**仅被一个课件节点引用**
