# 社区课件无法打开问题修复报告

## 问题描述
**报告时间**: 2026-04-12 20:01  
**现象**: 网站部署成功后,社区课件仍然无法打开

## 根本原因
GitHub Actions 部署 workflow 中**遗漏了 `community/` 目录**,导致:
- `community/index.json` 未被部署到 GitHub Pages
- 前端 `community-loader.js` 无法加载课件索引
- 82个社区课件全部不可访问

## 关键代码路径

### 前端加载逻辑
```javascript
// scripts/community-loader.js:14
const COMMUNITY_INDEX_URL = `https://raw.githubusercontent.com/${COMMUNITY_REPO}/main/community/index.json`;
```

### 部署配置(修复前)
```yaml
# .github/workflows/deploy-pages.yml
# 2. 复制必需目录
[ -d scripts ] && cp -r scripts _site/
[ -d examples ] && cp -r examples _site/
[ -d gallery ] && cp -r gallery _site/
[ -d docs ] && cp -r docs _site/
# ❌ 缺少 community/
```

## 修复方案

### 修改内容
添加一行复制 `community/` 目录的命令:

```yaml
[ -d community ] && cp -r community _site/
```

### 完整修复后的代码
```yaml
# 2. 复制必需目录
[ -d scripts ] && cp -r scripts _site/
[ -d styles ] && cp -r styles _site/
[ -d examples ] && cp -r examples _site/
[ -d gallery ] && cp -r gallery _site/
[ -d docs ] && cp -r docs _site/
[ -d community ] && cp -r community _site/  # ✅ 新增
```

## 后续建议

### 1. 添加部署验证脚本
创建 `scripts/verify-deployment.sh` 检查必需文件:
```bash
#!/bin/bash
REQUIRED_FILES=(
  "index.html"
  "manifest.json"
  "courseware-registry.json"
  "community/index.json"
)

for file in "${REQUIRED_FILES[@]}"; do
  [ -f "_site/$file" ] || echo "❌ Missing: $file"
done
```

### 2. 完善 workflow 注释
在 workflow 中明确列出所有必需目录:
```yaml
# 2. 复制必需目录
# - scripts: 前端逻辑(课件加载器等)
# - examples: 官方课件源文件
# - gallery: 课件展示画廊
# - community: 社区课件索引(index.json)
# - docs: 文档(可选)
```

### 3. 添加 CI 检查
在 workflow 中添加验证步骤:
```yaml
- name: Verify required files
  run: |
    test -f _site/community/index.json || exit 1
    echo "✅ All required files present"
```
