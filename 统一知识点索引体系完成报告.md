## 🎯 统一编号规则

### kp_id 格式
```
kp-{subject}-{stage}-{domain}-{concept}
```

**示例**：
```
kp-math-m-function-linear-func       # 数学·初中·函数·一次函数
kp-chn-e-pinyin-compound-vowel       # 语文·小学·拼音·复韵母
kp-phy-h-mechanics-newton-laws       # 物理·高中·力学·牛顿定律
kp-chem-m-reaction-oxidation         # 化学·初中·反应·氧化还原
```

### 学科代码（Subject Codes）
| 中文 | 代码 | 英文 |
|:---|:---|:---|
| 数学 | `math` | Mathematics |
| 物理 | `phy` | Physics |
| 化学 | `chem` | Chemistry |
| 生物 | `bio` | Biology |
| 语文 | `chn` | Chinese |
| 英语 | `eng` | English |
| 地理 | `geo` | Geography |
| 历史 | `hist` | History |

### 学段代码（Stage Codes）
| 中文 | 代码 | 年级 |
|:---|:---|:---|
| 小学 | `e` | Elementary (1-6) |
| 初中 | `m` | Middle (7-9) |
| 高中 | `h` | High (10-12) |

## 🛠️ 工具脚本（已实现）

### 1. `build-kp-index.py` — 主索引生成器
**功能**：
- 扫描所有知识树文件（`data/trees/*.json`）
- 提取所有知识点节点
- 自动生成符合规范的 `kp_id`
- 按字母顺序排列
- 去重并合并重复节点

**用法**：
```bash
python3 scripts/build-kp-index.py
```

**输出**：
- `data/knowledge-points/index.json` — 主索引
- `data/knowledge-points/migration-map.json` — 迁移映射表

---

### 2. `validate-kp-index.py` — 索引验证器
**功能**：
- 验证 `kp_id` 格式是否符合规范
- 检查重复ID
- 检查是否按字母顺序排列
- 验证关联关系（prerequisites, courses）的有效性
- 检查必填字段完整性

**用法**：
```bash
python3 scripts/validate-kp-index.py
```

**验证项**：
- ✅ ID格式：`kp-{subject}-{stage}-{domain}-{concept}`
- ✅ 无重复ID
- ✅ 按字母顺序排列
- ✅ prerequisites 引用的知识点存在
- ✅ courses 引用的课件文件存在
- ✅ 必填字段完整（kp_id, name_zh, subject, stage, status）

---

### 3. `query-kp.py` — 知识点查询工具
**功能**：
- 按 `kp_id` 查询
- 按旧 `node_id` 查询
- 按学科+学段查询
- 按课件ID查询
- 按名称关键词查询

**用法示例**：
```bash
# 按ID查询
python3 scripts/query-kp.py --id kp-math-m-function-linear-func --verbose

# 按旧ID查询（迁移时用）
python3 scripts/query-kp.py --old-id linear-function

# 查询某学科学段的所有知识点
python3 scripts/query-kp.py --subject math --stage m --count 50

# 查找包含某课件的知识点
python3 scripts/query-kp.py --course math-linear-function

# 按名称关键词搜索
python3 scripts/query-kp.py --name "一次函数" --verbose
```

## 📐 四层同步机制

### 1️⃣ 主索引（Master Index）
**位置**：`data/knowledge-points/index.json`
**作用**：唯一信源（Single Source of Truth）
```json
{
  "kp_id": "kp-math-m-function-linear-func",
  "name_zh": "一次函数",
  "subject": "math",
  "stage": "m",
  "grade": 8,
  "domain": "function",
  "courses": ["math-linear-function"],
  "status": "active"
}
```

### 2️⃣ 课件清单（Courseware Manifest）
**位置**：`examples/{course-id}/manifest.json`
**修改方案**：
```json
{
  "id": "math-linear-function",
  "kp_id": "kp-math-m-function-linear-func",  // ← 新增字段
  "node_id": "linear-function",               // ← 保留用于兼容
  "name": "一次函数",
  "subject": "math",
  "grade": 8
}
```

### 3️⃣ 知识树（Knowledge Tree）
**位置**：`data/trees/{subject}-{stage}.json`
**已完成**：节点 `id` 现在对应 `old_node_id`，未来逐步迁移为 `kp_id`
```json
{
  "id": "linear-function",  // ← 当前（未来改为 kp_id）
  "name": "一次函数",
  "courses": ["math-linear-function"],
  "status": "active"
}
```

### 4️⃣ 注册表（Registry）
**位置**：`registry.json`
**已修复**：`rebuild-index.py` 现在保留所有字段
```json

  "subject": "math",
  "grade": 8,
  "status": "official",  // ← 已修复（之前丢失）
  "path": "examples/math-linear-function"
}
```

## ✅ 已解决的问题

### 问题1：官方课件消失
**根因**：`rebuild-index.py` 重建索引时丢失了 `status` 字段
**修复**：
- 从备份恢复 9 个官方课件的 `status: "official"`
- 修复 `rebuild-index.py`，重建时保留 `status` 和其他重要字段
- 前端 `unified-loader.js` 按 `status === 'official'` 筛选官方课件

**结果**：✅ 9 个官方课件恢复显示

### 问题2：知识点ID不一致
**根因**：
- 课件 manifest 用 `node_id`（不含学科前缀）
- 课件目录名用 `course_id`（含学科前缀）
- 知识树节点用另一套 ID
- 没有统一的映射关系

**修复**：
- 建立主索引（`index.json`）作为唯一信源
- 生成迁移映射表（`migration-map.json`）
- 新知识点统一使用 `kp_id` 格式

**结果**：✅ 680 个知识点全部规范化，ID唯一且可追溯

### 问题3：重复节点
**根因**：手动编辑知识树时产生重复
**修复**：
- `build-kp-index.py` 自动去重
- 保留课件最多的版本
- 删除了大量重复节点（biology-middle: 20个，math-high: 15个，physics-middle: 6个）

**结果**：✅ 主索引无重复
