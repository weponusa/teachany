## 🐛 问题描述

用户反馈首页 Hero 区域显示的课件统计数字不正确:

- **知识节点**: 646个 ✅

### 实际正确数据
- **官方课件**: 8个 ✅
- **社区课件**: 82个 ✅
- **学科**: 8个 ✅ (math, biology, physics, history, geography, chemistry, chinese, custom)
- **知识节点**: 646个 ✅

### 差异分析
| 项目 | 错误值 | 正确值 | 差异 | 原因 |
|:---|:---:|:---:|:---:|:---|
| 官方课件 | 86 | 8 | -78 | 硬编码旧数据(v2.1重构前) |
| 社区课件 | 92 | 82 | -10 | 硬编码旧数据 |
| 学科数 | 6 | 8 | +2 | 未计入 chinese 和 custom |

## 🔍 根本原因

### 1. 硬编码静态HTML
`index.html` 第531-544行硬编码了统计数字:

```html
<div class="stats">
  <div class="stat">
    <div class="stat-num">86</div>  <!-- ❌ 硬编码 -->
    <div class="stat-label">官方课件 Official</div>
  </div>
  <div class="stat">
    <div class="stat-num">92</div>  <!-- ❌ 硬编码 -->
    <div class="stat-label">社区课件 Community</div>
  </div>
</div>
```

### 2. 缺少动态更新机制
- `registry-loader.js` 有 `updateStats()` 函数,但只更新包含 "course" 关键词的统计项
- `community-loader.js` 完全没有统计更新逻辑
- 两个加载器相互独立,无法汇总统计

### 3. 浏览器缓存
- 两个加载器都有 5分钟的 localStorage 缓存(CACHE_TTL)
- 用户可能看到的是缓存的旧数据

### 核心思路
1. **统一状态管理**: 创建全局 `window.TeachAnyStats` 对象

3. **自动更新**: 统计模块自动计算并更新 DOM
4. **学科去重**: 使用 `Set` 自动去重官方和社区的学科

### 2. scripts/stats-updater.js (新模块)

**功能**:
- 全局状态管理 (`window.TeachAnyStats`)
- 注册官方课件数据 (`registerOfficial`)
- 注册社区课件数据 (`registerCommunity`)
- 自动更新 DOM 元素 (`updateStatsDisplay`)
- 自动去重学科 (`Set`)

**核心代码**:
```javascript
window.TeachAnyStats = {
  officialCount: 0,
  communityCount: 0,
  subjects: new Set(),
  nodesCount: 646
};

function updateStatsDisplay() {
  document.getElementById('officialCountStat').textContent = stats.officialCount;
  document.getElementById('communityCountStat').textContent = stats.communityCount;
  document.getElementById('subjectsCountStat').textContent = stats.subjects.size;
}

window.TeachAnyStats.registerOfficial = function(registry) {
  stats.officialCount = registry.courses.length;
  registry.courses.forEach(c => stats.subjects.add(c.subject));
  updateStatsDisplay();
};

window.TeachAnyStats.registerCommunity = function(index) {
  stats.communityCount = index.courses.length;
  index.courses.forEach(c => stats.subjects.add(c.subject));
  updateStatsDisplay();
};
```
